UT Coding Projects:

The following is a list of coding projects I did at the University of Texas for my CS courses there. Due to policies at UT I cannot publically put up my code, but I will go into detail on some of the projects that stood out to me.

Huffman Coding Compression:

This project was for CS 314: Data Structures. In this project I had to use binary trees, and queues to do huffman encoding to compress files. The program would read a byte at a time, and count up the number of occurences of each byte in the file. The tally for each byte along with the type of byte would be placed into a priority queue. They would then be popped from the queue in order to make a binary tree. Traversing this tree would determine the new byte value for whatever node it arrived at. The new values were then written out.

Djikstras Shortest Path Algorithm

This project wasa for CS 314: Data Structures. In this project, I rewrote Djikstra's shortest weighted and unweighted path algorithm. This algorithm was then used to find the shortest paths in a graph.

Bomb Lab

This project was for CS 429: Computer Architecture. In this lab I was assigned a "Bomb," which was really just an assembly program with 6 phases. For each phase, I had to read, debug, and deconstruct the assembly code to figure out what to input in order to "defuse" the "bomb." In the last phase I also had to read through obfuscated code to determine what to pass it.

Attack Lab

This project wasa for CS 429: Computer Architecture. In this lab I passed a string to a program, and intentionally overflowed the input in order to get the program to execute code it hadn't intended on executing. For the first 3 phases of this lab I was able to write in code which could be executed, and call said code. For the last 2 phases I had to find sections of code within the program to call in order to do certain things.

This project heavily used quaternion rotations to not only deal with rotating the player, but also placing them in the proper location after walking through the portals. Procedural Terrain Generation

This project generates a random landscape using a perlin noise map. The black/white values of the noise map are translated into height, which is then stored into a 2D array. This 2D array of heights is used to generate a 3D mesh of the terrain. A texture is then applied by selecting colors for certain ranges of heights. The texture is then applied on to the mesh, creating the landscape the player sees. Click the link below to go to the itch.io page to download the project.
